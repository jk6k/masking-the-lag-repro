# HPAT Parameterized Model Description

This file explains the HPAT-style parameterized model used under `hpat_model/`, including its inputs, assumptions, outputs, and extension points.

This is an estimation model rather than a direct hardware measurement. It currently covers Conv2d and Linear layers, attention `QK/AV`, and configurable electronic-domain estimates for softmax, normalization, and activation.

## Purpose

The model estimates MobileViT inference latency and energy using configurable photonic-hardware parameters when no real photonic prototype is available.

## Inputs

1. operator files generated by `extract_model_ops.py`
2. configuration files such as `hpat_config.yaml`

## Outputs

- model-level summaries in `results/hpat_estimates.csv`
- layer-level detail files in `results/hpat_ops/*_ops.csv`

The summary includes breakdown terms such as loading energy, detection energy, laser energy, memory energy, static energy, and elementwise electronic costs.

## Computation Flow

At a high level, the model:

1. reads layer shapes from `ops_*.json`
2. tiles the computation according to photonic array parameters
3. estimates cycles, latency, and energy per layer
4. aggregates the totals across the model

## Important Parameters

Photonic configuration includes:

- `tile_k`
- `tile_k_prime`
- `cycles_per_tile`
- `sample_rate_gsps`

Energy configuration includes:

- `dac_power_mw`
- `adc_power_mw`
- `modulator_power_mw`
- `pd_power_mw`
- `tia_power_mw`
- `laser_power_mw`
- `mem_energy_pj`
- `static_power_mw`

## Conceptual Formulas

The implementation lives in `hpat_model.py`. Conceptually:

- tile counts are computed from the GEMM dimensions and tile sizes
- latency is derived from total cycles and sample rate
- energy is the sum of load, detect, laser, memory, and static terms

## Current Scope

Included:

- Conv2d and Linear GEMM
- attention `QK^T` and `AV`
- elementwise electronic-domain estimates
- DAC, ADC, modulation, PD, TIA, laser, memory, and static terms

Not included:

- on-chip and off-chip communication hierarchy
- extra digital-control overhead beyond the modeled terms

## How to Extend

- adjust attention or electronic-domain costs through the configuration files
- modify device assumptions in `hpat_config.yaml`
- extend formulas and energy terms in `hpat_model.py`

## Quick Run

```bash
python3 hpat_model/extract_model_ops.py --out_dir hpat_model/ops
python3 hpat_model/run_hpat_model.py --ops_dir hpat_model/ops --out results/hpat_estimates.csv
```

ASIC single-configuration sweep:

```bash
python3 hpat_model/run_hpat_sweep.py \
  --ops_dir hpat_model/ops \
  --config hpat_model/hpat_config_asic.yaml \
  --out results/hpat_estimates_range.csv \
  --save_per_config
```
